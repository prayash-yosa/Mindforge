Summary

Background processing pipeline picks pending syllabus documents, extracts text (PDF via pdf-parse, images via OCR placeholder), calls OpenAI-compatible LLM for structured concept extraction, and stores results as lesson_session entities with graceful fallback.

Processing Pipeline

  1. Text Extraction: PDF → pdf-parse; Plain text → direct read; Image → OCR placeholder
  2. AI Analysis: System prompt requests JSON with conceptSummary, learningObjectives[], hasNumericals, chapters[], topics[]; temperature 0.2; configurable timeout
  3. Fallback: When no API key or AI fails — regex-based numerical detection + generic objectives
  4. Storage: lesson_session created; raw text stored (10K truncation); document status updated

AI Prompt Template

System: You are an educational content analyst. Analyze the
following syllabus/lesson content for the subject "{subject}"
and return a JSON object with:
- "conceptSummary": 2-4 sentence summary
- "learningObjectives": array of 3-7 objectives
- "hasNumericals": boolean
- "chapters": array of chapter names
- "topics": array of topics covered
Return ONLY valid JSON, no markdown.
