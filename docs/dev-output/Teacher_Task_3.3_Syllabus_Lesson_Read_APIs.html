Teacher Task 3.3 — Syllabus & Lesson Read APIs for Frontend

Sprint: 3 — Syllabus & AI Ingestion | App: Teacher | Status: Done | Date: 2026-03-02 | SP: 3

Summary

Read APIs for listing syllabus uploads with processing status, retrieving AI-extracted lesson details (concept summaries, learning objectives, chapters, topics, has_numericals flags), and individual document/lesson lookups. Failed processing states are clearly surfaced.

Acceptance Criteria

#CriterionStatusImplementation
1List uploads with status (Processing, Ready, Failed)DoneGET /v1/syllabus/class/:classId
2Get chapters/topics and lesson details with flagsDoneGET /v1/syllabus/lessons/class/:classId?subject=
3Failed states clearly surfaced for UXDoneerrorMessage field on failed documents

API Endpoints

MethodPathPurpose
GET/v1/syllabus/class/:classIdList all syllabus uploads for a class
GET/v1/syllabus/:docIdGet single document details
GET/v1/syllabus/:docId/lessonGet lesson from specific document
GET/v1/syllabus/lessons/class/:classIdList all lessons for a class
GET/v1/syllabus/lessons/:lessonIdGet single lesson details

Response: SyllabusListItem

FieldTypeNotes
idstring (UUID)Document ID
classIdstringParent class
subjectstringSubject name
fileNamestringOriginal file name
statusstringpending / processing / ready / failed
errorMessagestring?Present when status = failed
uploadedAtDateUpload timestamp

Response: LessonDetail

FieldTypeNotes
idstring (UUID)Lesson ID
conceptSummarystringAI-generated summary
learningObjectivesstring[]Parsed from JSON
hasNumericalsbooleanNumerical content flag
chaptersstring[]Parsed from JSON
topicsstring[]Parsed from JSON

Service Layer

SyllabusReadService provides:

  • listByClass(classId) — ordered by uploadedAt DESC
  • getDocumentById(docId) — 404 if missing
  • getLessonsByClass(classId, subject?) — optional subject filter
  • getLessonById(lessonId) — 404 if missing
  • getLessonByDocument(documentId) — null if not processed

Verification

  • Build: npx nest build — zero errors
  • Boot: All 5 read routes mapped at startup
  • Response: Consistent { success: true, data: ... } wrapper
  • Error states: Missing entities → 404; failed docs include errorMessage
