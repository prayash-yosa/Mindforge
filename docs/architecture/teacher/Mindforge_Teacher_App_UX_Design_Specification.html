1. UX System Overview

Web-first teacher experience with responsive mobile layouts, aligned with the existing Mindforge brand and student app.

UX Principles

  • Teacher-first, time-respectful – attendance and test flows are 1–2 clicks.
  • Web-first, mobile-responsible – primary layout is desktop; key flows work on phone.
  • Data clarity over decoration – numeric KPIs and trends are easy to scan.
  • Consistency with Student app – same sage-green / cream palette and tone.
  • AI as assistant – teacher keeps control over syllabus and assessments.

2. Key User Flows

2.1 Take Attendance for a Class Session

Mark attendance quickly for each class, with default Present and edit window. 

Desktop:
Dashboard → Today’s timetable → "Take attendance"
Classes & Attendance:
  • Select Class 8A • Math and date
  • All students default Present
  • Toggle Absent where needed; add notes
  • Save attendance → success toast
  • Optional: open calendar view

Mobile:
Dashboard (phone) → "Take attendance" card
  • Scrollable Present/Absent list
  • Save attendance → "View calendar"

2.2 Upload Syllabus and Inspect AI Concepts

Sidebar → Syllabus
  • Choose class/subject
  • Upload PDF/Image
  • See upload status (Processing/Ready/Failed)
  • When Ready: browse Extracted Concepts
       Chapters → Topics → details:
       - Summary, Learning objectives
       - Flags (Has numericals: Yes/No)

2.3 Configure Tests (Online & Offline)

Sidebar → Tests

Online Daily Quiz:
  • Select Class / Subject / Chapter
  • Set question count & marks per question
  • "Enforce NO numericals (online)" = ON
  • Preview generated objective questions
  • Save & Publish

Offline Printable Test:
  • Define question mix, marks, duration
  • Generate PDFs:
       - Student Paper
       - Teacher Key (with solutions)

2.4 View Results & Analytics

Sidebar → Results & Analytics
  • Pick Class, Subject, Period
  • View KPIs: Average Score, Attendance %, etc.
  • Inspect trend charts
  • Scroll to "Students Needing Attention"
  • Open student details from table

4. Screen Specifications (Desktop + Mobile)

4.1 Dashboard

Overview of today’s workload and issues.

  • KPI cards: Today’s Classes, Attendance issues, Pending test evaluations.
  • Today’s timetable: upcoming classes with Take attendance actions.
  • Class attendance summary: per-class donuts (Present/Absent %).
  • Recent alerts: absence streaks, missed tests, auto-submits.
Dashboard — Desktop & Mobile

4.2 Classes & Attendance

Per-session attendance with fast Present/Absent toggles.

  • Class/date/session selection row with edit window label.
  • Summary chips: Present count, Absent count, Editable until.
  • Student table or list with Present/Absent pill and Notes field.
  • Mini calendar for this class; button to open full attendance calendar.
  • Primary actions: Save attendance, Mark all present, Undo last change.

4.3 Syllabus Upload & Concepts

Turn raw syllabus documents into structured lessons.

  • Upload dropzone with supported formats (PDF/JPEG/PNG).
  • Recent uploads pane with Processing / Ready / Failed statuses.
  • Extracted Concepts list: Chapters and Topics with expand/collapse.
  • Topic detail panel: summary text, learning objectives, flags (Has numericals).

4.4 Tests (Online & Offline)

Configure daily quizzes and printable tests from syllabus.

  • Online Daily Quiz: objective-only, “Enforce NO numericals (online)” toggle.
  • Quiz details form: class/subject/chapter, question count, marks.
  • Question preview panel listing MCQ/T&F/FIB items.
  • Offline Printable Test: controls for question mix, marks, duration, PDF generation.

Summarise performance and attendance; highlight at-risk students.

  • Filter bar: Class, Subject, Period.
  • KPI cards for average score, attendance %, students at risk, tests this week.
  • Charts: score over time and attendance by week.
  • “Students Needing Attention” table with weak topics and quick actions.

5. Accessibility Notes

  • Keyboard navigation: all sidebar items and primary actions must be operable by keyboard.
  • Attendance table: clear focus states; support quick toggling via keyboard.
  • Color contrast: maintain WCAG 2.1 AA contrast for text and UI elements.
  • Status messages: use role="status" / role="alert" for save and error states.
  • Charts: provide text summaries (“Average score 82%, up 2% from last month”).
